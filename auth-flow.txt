
Все запросы от авторизованного пользователя
	ЗАГОЛОВОК - в headers -  “Authorization”:  "Bearer {jwtToken}"
		(jwtToken - токен, полученный из успешного ответа авторизации)
		для этого - необходимо использовать axios interceptors

	

Поведения при авторизации
	Все запросы включают лоадер

	ИНПУТЫ (Общее)
		Плейсхолдер - по центру. При ФОКУСЕ/НЕПУСТОМ - плейсхолдер расположен выше
		Валидация - ОНБЛЮР / КЛИК на отправку формы 
			Незаполненное (+ подсказка)
	ИНПУТ (пароль)
		при ВВОДЕ - (по дефолту) символы маскируются, глаз закрыт
		КЛИК на глаз - глаз открыт, пароль без маски

	ВХОД
		Логин, пароль - ОБЯЗАТЕЛЬНЫЕ
		Введенные данные не валидируются
		Сабмит - отправка запроса POST /api/auth/local
		Ответ на сабмит - получение ТОКЕНА (его исп. при дальнейших запросах)
		Ответ с ошибкой - (400) инпуты подсвечены, подсказка "неверно", введенные значения остаются, появляется поле "восстановить"
			*другие статусы - отображение экрана "что-то пошло не так"

Авторизация
	Последующие открытия - редирект на главную
			

РЕГИСТРАЦИЯ
	Попадание - по прямой ссылке \ КЛИК на "регистрация" на входе
	Все поля ОБЯЗАТЕЛЬНЫЕ
	Элементы:
		1. Номер шага
		2. поля для заполнения
		3. подсказки
		4. кнопка Перехода шага (+валидация на текущем шаге)
		5. переход на ВХОД
	Требования к валидации
		1 ШАГ
			ЛОГИН - латинск. И цифры
				Валидация - по мере ввода (динамическая)
				ПРИ ФОКУСЕ - в подсказке подсвечивается невыполненные требования (отдельно)
				при ОНБЛЮР\САБМИТ - подсветка всего текста
			ПАРОЛЬ - не менее 8, минимум 1 заглавная, минимум 1 цифра
				Валидация - по мере ввода (динамическая)
				ПРИ ФОКУСЕ - в подсказке подсвечивается невыполненные требования (отдельно)
				при ОНБЛЮР\САБМИТ - подсветка всего текста
				ЕСЛИ УСПЕШНО проходит динамическую валидацию - ГАЛОЧКА возле глаза
		2 ШАГ
			ИМЯ, ФАМИЛИЯ - только на ПУСТОЕ

		3 ШАГ
			ТЕЛЕФОН	
				только цифры, код оператора из списка 29|33|25|44
				Вводить можно только цифры
				МАСКА (можно библиотеки, пример - react-text-mask)
				Несоответствие маске (длинна) - подсветка + подсказка
			ПОЧТА
				Соответсвие базовым требованиям (исп. регулярное выражение)

	МАСКИ - для пароля, телефона
		ТЕЛЕФОН - +375 (xx) xxx-xx-xx, где x - только цифры
			Пользователь начинает вводить с кода оператора
			Код +375 (и остальные символы в маске) подставляется автоматически
	
	КЛИК на СЛЕДУЮЩИЙ ШАГ - доп. валидация

	ФИНАЛ
		сабмит, отправка данных через запрос POST /api/auth/local/register
		Успешный ответ - экран успеха, кнопка на редирект на ВХОД
		ОШИБКА
			(400) - логин или пароль существуют, НАЗАД к регистрации (переход к 1 шагу с начала - очистка всех полей)
			(др.) - экран ДАННЫЕ не сохранились, кнопка повторного запроса



ВОССТАНОВЛЕНИЕ ПАРОЛЯ	
	ПОПАДАЕМ (только для НЕАВТ. пользователей)
		- по прямой ссылке
		- по нажатию на "ВОССТАНОВИТЬ?" \ "ЗАБЫЛИ ЛОГИН ИЛИ ПАРОЛЬ?" на экране АВТОРИЗАЦИИ
	
	Ввести почту (зарегистрированного пользователя)
		Валидация - ПУСТОЕ / та же регулярка для почты

	Клик на ВОССТАНОВИТЬ
		отправка запроса POST  /api/auth/forgot-password
		УСПЕХ - экран успеха (письмо выслано)
		ОШИБКА - почта не чистится, ошибка в подсказке
	
	*на почту отправляется ссылка*
		Имеет структуру + КОД (CODE) - http://localhost:3000/#/forgot-pass?code=CODE

	Переход по ссылке - экран ВОССТАНОВЛЕНИЕ
		Валидация паролей - как на Регистрации (НО - отличия)
			Подсказка на валидацию пароля + ГАЛОЧКА - только у первого пароля
			Под вторым инпутом - подсказка на СОВПАДЕНИЕ
			ПРИ НЕСОВПАДЕНИИ - кнопка ДИЗЕЙБЛ
		КЛИК на СОХРАНИТЬ изменения - отправка запроса POST /api/auth/reset-password.
			УСПЕХ - ЭКРАН успеха + кнопка на ВХОД
			ОШИБКА - ЭКРАН Данные не сохранились, Переход на экран создания паролей


ВЫХОД из АККАУНТА

	КЛИК на ИКОНКУ профиля - открыть выпадающее меню (Профиль, Выход)
